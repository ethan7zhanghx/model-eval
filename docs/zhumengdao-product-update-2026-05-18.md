# 筑梦岛产品更新说明（2026-05-18）

本文档记录 2026-05-18 筑梦岛相关功能的最终产品变化，仅保留面向测试和使用的功能结果。

## 一、对话配置能力

### System Prompt 可配置

筑梦岛设置中新增 System Prompt 配置入口，测试人员可以在前端直接调整本轮对话使用的系统提示词。

为保证评测记录一致性，一旦会话开始，角色、温度和 System Prompt 会锁定在当前会话内，不支持中途变更。后台对话详情中会展示本轮会话实际使用的 System Prompt，便于复盘。

### 新增置顶角色

角色列表新增并置顶 4 个人工写入角色，方便测试人员优先选择：

- 凌意
- 何最
- 裴君肆
- 程许

角色数据补充了性别字段，并在角色预览和对话 Prompt 中使用。

## 二、灵感模式

### 手动触发灵感候选

灵感模式从“自动生成”调整为“手动触发”。用户在正常 A/B 对话中选择胜者后，对话输入区会出现“灵感”和“继续聊”两个操作按钮。

点击“灵感”后，系统会基于上一轮已选中的角色回复，生成 4 条用户回复候选。候选由 A/B 两个模型分别生成，并随机混排展示，避免测试人员通过位置判断模型来源。

### 候选选择与编辑

用户选择灵感候选后，候选内容会填入输入框。用户可以直接发送，也可以在发送前继续编辑。

系统会记录：

- 本次灵感候选来自哪个模型
- 用户选择了哪条候选
- 用户最终发送的内容
- 用户是否完全手写回复

候选在发送前不展示模型来源，发送后再在记录中展示，保证盲评流程不被提前影响。

### 灵感记录独立保存

灵感生成被视为独立记录，不混入正常对话 A/B 统计。后台对话详情中会展示本次灵感候选、用户选择和最终发送内容。

灵感记录保存流程做了流畅度优化：选择候选后，即使记录保存仍在进行，也不会阻塞用户发送。

## 三、继续聊模式

### 手动触发继续聊

新增“继续聊”功能。用户在正常 A/B 对话中选择胜者后，如果暂时不想自己回复，可以点击“继续聊”，让 AI 基于当前上下文继续以角色身份多输出一轮。

继续聊同样采用 A/B 盲选流程：

- A/B 两个模型各生成一条角色续聊内容
- 展示顺序随机
- 用户选择后，所选内容直接进入对话历史
- 后台独立记录继续聊轮次和胜负

### 会话恢复支持

历史会话恢复时，灵感模式和继续聊模式的记录会被正确还原，不再显示为空消息或异常内容。

## 四、数据后台

### 统计页分为三类 A/B

后台数据面板按评测类型拆分为三个 Tab：

- 正常对话 A/B
- 灵感模式 A/B
- 继续聊 A/B

三类数据分别统计，避免不同任务类型混在同一个胜率口径里。

### 灵感模式统计升级

灵感模式后台改为与正常对话、继续聊一致的 A/B 看板结构，包括：

- 总生成数
- 已选择数
- A/B 胜场
- 平均 TTFT
- 平均 TPS
- 模型排名
- 对战矩阵
- 速度统计

未被用户采用的灵感生成不会计入胜负统计，但仍会保留在对话详情中，便于查看候选质量。

### 对话详情展示优化

后台对话记录详情对不同生成类型做了区分展示：

- 正常对话
- 灵感候选
- 继续聊内容

灵感候选卡片布局收紧，减少空白和信息割裂感。用户最终采用的候选会突出显示，并展示最终发送内容。

### 模型排名条形图修正

模型排名仍使用 Bradley-Terry 分数决定排序，但右侧横向条改为展示实际胜率比例。

例如 `70%` 胜率会显示为约 70% 长度，不再因为排名第一而显示满格。排名行也改为固定列布局，保证所有条形图左侧对齐。

## 五、体验与稳定性调整

### 辅助按钮展示优化

正常 A/B 选择完成后，输入区会直接展示“灵感”和“继续聊”按钮，不再显示额外说明文字。

### 生成 Prompt 放松

灵感和继续聊的生成 Prompt 去除了过多风格、长度和表达限制，只保留角色扮演对话的基础要求。

灵感模式保留最小 JSON 输出格式，确保系统可以稳定解析两条候选。灵感和继续聊均保留基础括号旁白规则，保证动作、表情、心理等内容的呈现符合当前角色扮演体验。

### 继续聊选择状态即时更新

修复继续聊选项点击后短暂停留在“未选择”状态的问题。用户选择继续聊候选后，前台会立即显示选中状态。
