# OpenRouter 多模型多轮 Case 对比台

一个用于模型评测的小工具：针对同一组多轮 Prompt，同时在多个模型上执行并对比结果，便于做横向评估和复盘。

## 项目功能介绍

- 多模型并行对比：每一列绑定一个模型，同一轮 Prompt 同时执行。
- 多轮流程执行：一次点击即可从第 1 轮跑到最后 1 轮，自动保留上下文对话历史。
- 全局 System Prompt：在整条评测流程中持续生效。
- 模型列表自动加载：从 OpenRouter 官方模型接口拉取，失败时自动回退缓存。
- Prompt 文件导入：支持 `.md/.txt`，自动切分为多轮任务。
- 结果矩阵展示：每个单元格展示返回文本、耗时和 token 使用情况。
- 成本预估：根据模型输入/输出单价，估算本次执行总花费。
- 配置历史管理：支持保存、载入、删除历史配置（按 workspace 隔离）。
- 数据导出：可导出完整 JSON（配置、结果、历史快照）。

## 页面说明

- 全局配置区：API Key、Temperature、Workspace、System Prompt、执行按钮。
- 模型列配置区：添加/删除模型列、选择模型、查看模型价格。
- 配置历史区：保存当前配置、回滚到历史配置。
- 轮次 Case 表：编辑多轮 Prompt、导入文件、查看每个模型的结果矩阵。

## 本地开发

要求：Node.js 18+

```bash
npm install
npm run dev
```

打开 `http://127.0.0.1:8080`

## 部署到 Vercel

### 基础部署（可直接用）

1. 将仓库导入 Vercel。
2. 保持默认设置直接 Deploy。
3. 部署完成后即可使用核心评测功能。

未配置 KV 时，历史配置会自动保存到浏览器本地存储（`localStorage`）。

### 开启服务端历史持久化（推荐，可选）

在 Vercel 项目中添加环境变量：

- `KV_REST_API_URL`
- `KV_REST_API_TOKEN`

然后重新部署。此时 `/api/config-history` 会把历史写入 Vercel KV，可跨设备共享历史空间。

## 使用流程

1. 填入 OpenRouter API Key。
2. 刷新并选择模型列。
3. 编写轮次 Prompt（或导入 `.md/.txt`）。
4. 点击“执行完整流程”。
5. 查看每轮每模型结果、耗时、token。
6. 需要时保存配置历史、导出 JSON。

## 注意事项

- 费用估算是近似值：输入 token 按历史累计估算，输出 token 按系数估算（每轮至少 24）。
- 浏览器会直接请求 OpenRouter，建议仅在可信环境使用。
- 若提示“未配置服务端持久化”，不影响评测执行，仅表示历史暂存于本地浏览器。
